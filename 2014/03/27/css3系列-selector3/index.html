CSS3系列-selector3

本篇文章所弹到的选择器均为CSS3中的选择器

选择器kill point

在CSS的选择器中，大家可能会觉得各种的错综复杂，现在我们就一起来捋一捋其中的规则和规律。
在CSS3关于选择器的规范中，对选择器的定义为：一个选择器是由一个或者多个简单选择器通过连接符而构成的。在这一句话我们要提取出两个关键词：“简单选择器”和“连接符”。
“简单选择器( simple selector)”在CSS3的规范中指定为：类型选择器( type selector), 全局选择器(universal selector), 属性选择器(attribute selector), 类选择器( class selector), ID选择器(ID selector), 或者伪类(pseudo-class)。
“连接符 (Combinators)”特指下面四种符号：大于符号(greater than sign)[u+003e, >], 加号(plus sign)[u+002b, +],波浪号(tide)[u+007e, ~]和空格。

深入CSS3选择器

通过w3c关于CSS3的规范中关于简单选择器的列表，我们不难分析出，在CSS3对简单选择器对变化中主要是针对属性选择器和伪类的。通过这些新增的选择器我们能够找到更多想要的内容。

属性选择器

如果说CSS2中属性选择器完成的是对一个元素有特定属性的选择，那么在CSS3中属性选择器更多的有了正则的成分在里面。在CSS3中属性选择器增加了类似正则里面“^”的开始选择符，”$”结尾选择符和“*”通配符。

E[foo^=”bar”]：以某个值开始的选择器。
在下面的例子中，我们通过开始属性选择符，选择出了属性以xiao开头的标签。

<div>
  <p shawn="xiaohero">This is test for xiao</p>
  <p shawn="shuanghero">This is test for shuang</p>
  <p shawn="xiongxiao">This is test for xiong xiao</p>
  <p shawn="xiaoxiong">This is test for xiao xiong</p>
</div>

See the Pen yeCkn by shawnxiao (@f2ecouple) on CodePen.


这个属性选择符在实际的应用中，我们可以用来对选择器进行分类，比如给元素区分位置信息，选择出相似的元素。

<div>
  <p data-log-position="PLUGIN_HISTORY_CLICK">PLUGIN_HISTORY_CLICK</p>
  <p data-log-position="PLUGIN_HISTORY_CLICK">PLUGIN_HISTORY_CLICK</p>
  <p data-log-position="BAR_HISTORY_CLICK">BAR_HISTORY_CLICK</p>
  <p data-log-position="POPUP_HISTORY_CLICK">POPUP_HISTORY_CLICK</p>
</div>

See the Pen yeCkn by shawnxiao (@f2ecouple) on CodePen.


E[foo$=”bar”]:以某个值结束的选择器。
这个和正则符中的结尾有异曲同工的作用，选择出属性以某个值结束的标签。
